HTML

  • The head element contains information (metadata) about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML attribute defines additional information about an element.
  • In semantic HTML elements, the name describes its content and the role it plays.
  • There are different heading levels (h1-h6) the smaller the number the larger the text.
  • p is used for a paragraph or block of text. Used to wrap text on webpages.
  • ul, ol, and li are for making lists and stand for unordered list, ordered list, and list item respectively.
  • a stands for anchor and is used to link a webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • HTML elements can be styled suing the elements as selectors or by using the class attribute as a selector.
  • Inline CSS is styled directly in the HTML file and is useful for quick and permanant changes but is less flexible.
  • Internal style sheets are entire stylesheet embeded directly in the HTML file usign a style element in the head element.
  • External style sheet is most common and is preferred because it keeps all CSS rules in a seperate file making design changes easier.
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • Do Not Repeat Yourself (DRY) means using or sharing pieces of code to reduce the number of lines and the overall complexity of a codebase.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • cd is change to directory
  • sl lists detailed directory contents
  • git add -A commits all files in the working branch
  • git commit -m associates a message with our commit
  • Git branching allows changes to be worked on seperately and assits with version control
